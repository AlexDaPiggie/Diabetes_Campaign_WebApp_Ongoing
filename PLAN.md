# Familiarization Plan

## Goal
Create a concise overview of the repository's structure and key files to quickly get up to speed.

## Steps
1. **Read README.md** – get high‑level description of the project.
2. **List top‑level files and directories** – identify main entry points (e.g., `index.html`, `AGENTS.md`, `Guidelines/`).
3. **Open `index.html`** – see what the web page renders and which assets it includes.
4. **Open `AGENTS.md`** – understand any custom Claude agents defined for the repo.
5. **Browse `Guidelines/`** – note design assets and any documentation (`DESIGN.md`).
6. **Summarize findings** – produce a short summary of the project purpose, front‑end entry point, and any notable docs.

## Success criteria
- A written summary covering the project purpose, main files, and where to find design assets.
- No files are modified.
